Interesting find with Entity Framework today. It likely applies to NHibernate as well.
[2/12/2014 4:43:44 PM] Mike Sigsworth: If you have a query that reuses the same navigation property more than once in the query or projection, use a "let" statement to create an inline variable.
[2/12/2014 4:46:21 PM | Edited 4:47:08 PM] Mike Sigsworth: For example:

from c in db.Customers
select new {
c.Name,
c.Address.City,
c.Address.State
}

This will cause two LEFT OUTER JOINs on the Address table to be generated in the SQL. But using the following code just generates one JOIN:

from c in db.Customers
let address = c.Address
select new {
c.Name,
address.City,
address.State
}
[2/12/2014 4:46:41 PM] Mike Sigsworth: Query performance will be dramatically improved.
